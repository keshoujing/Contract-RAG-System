# Contract Q&A Page Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build the Contract-RAG Q&A page and source verification popup from the approved Pencil design.

**Architecture:** Reuse the existing agentic `POST /api/query` endpoint and render its polymorphic `evidence[]` on one React page. Add a small backend endpoint for archived contract page PNGs so the frontend can show a precise single-page verification modal with bbox highlights.

**Tech Stack:** FastAPI, PyMuPDF rendering helpers, React, TypeScript, React Query, React Router, Vitest, Testing Library.

Define:

```ts
export interface QueryRecordEvidence {
  kind: "record";
  contract_id: string;
  title?: string;
  fields: Record<string, string | number | null | undefined>;
}

export interface QueryClauseEvidence {
  kind: "clause";
  contract_id: string;
  page?: number;
  section?: string;
  snippet: string;
  bbox?: [number, number, number, number] | number[];
}

export type QueryEvidence = QueryRecordEvidence | QueryClauseEvidence;

export interface QueryResponse {
  question: string;
  answer: string;
  evidence: QueryEvidence[];
}
```

Add:

```ts
export async function askQuestion(payload: { question: string; contract_id?: string | null }): Promise<QueryResponse>
```
